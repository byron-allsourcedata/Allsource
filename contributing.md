# Frontend

_ПРАВИЛА АКТУАЛЬНЫ КРОМЕ ДНЯ ДЕМО_

1. Жесткая типизация(никаких any никогда и нигде(кроме исключительных случаев)).
2. DRY (Don`t repeat yourself) больше двух раз повторяется - выноси в компонент.
3. Все стили выносим из основного компонента. Внутри компонента только верстка(без стилей) и функциональная логика.
4. Желательный размер компонента - плюс минус от 300 до 500(Больше 1к карается).
5. Для всех zIndex использовать ENUM.

# Backend

1. Типизация для всего.
2. Для объектов использовать из Pydantic - BaseModel.
3. Не разрастать огромные файлы.
4. Вместо Depends, добавляем к service и persistance @injectable.
5. Код скриптов должен быть сервисом.
6. Если нужно сохранить какой то json то сохраняем в отдельную таблицу с привязкой.
7. Нужно использовать консистентные названия везде (роут должен быть согласован с сервисами, а сервисы с методами)
